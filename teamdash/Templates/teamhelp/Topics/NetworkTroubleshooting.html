Troubleshooting Network Connectivity

The Team Dashboard provides many services to team members, such as the ability to join a team project or view the team project plan. These services are generally reached by pointing your Web browser at an appropriate URL.

If you are unable to view a URL given to you by your team leader, try the steps below to resolve the problem.

  1. Contact your team leader and ensure that (a) the Team Dashboard is running, and (b) the computer running the Team Dashboard is connected to the network. If your team leader verifies that the Team Dashboard is running and its computer is connected to the network, but you still cannot connect, continue to the next step.
  2. Take a look at the URL you are attempting to visit. It will look something like this: 
    http://team-computer:3000/...
    The URL will begin with the characters "http://" and will then have the name of your team leader's computer, followed by a colon (:) and a number. The team computer name may include several words separated by periods (.). Copy down this team computer name, as the steps below will require it.
  3. Many organizations require their employees to use a proxy server when accessing the Internet. If your Web browser attempts to use a proxy server to contact your team leader's computer, it is likely to fail. Ensure that your Web browser is properly configured.
    • If you are using Internet Explorer, choose Tools → Internet Options. Select the Connections tab and click the LAN Settings button. Then ensure that the Bypass proxy server for local addresses box is checked.

    • If you are using Netscape or Firefox, choose Tools → Options. Open the Advanced → Network preference page, click on the Connection Settings button, and add the team computer name to the No Proxy for: field.

    After making this configuration change in your browser, try refreshing the page in your browser. If it still does not appear, continue to the next step.
  4. To determine whether your computer can contact the team leader's computer, open up a command prompt window, and type the following command: 
    ping team-computer
    replacing the highlighted letters with the name of the team computer you copied from the team project URL. If you see the message, 
    Unknown host team-computer
    then your computer is having trouble finding the IP address of the team computer. You may be able to work around the problem by using an IP address directly. Contact your local support staff and ask them to help you look up the IP address of your team leader's computer - it will be a number like 10.20.30.40. Then edit the URL you are trying to visit, replacing the computer name with the IP address: 
    http://10.20.30.40:3000/...
    If the page still does not appear, continue to the next step.
  5. If none of the above approaches work, your organization's network may be configured to block peer-to-peer traffic. To work around the problem, each team member can take these steps:
    • Install the Process Dashboard, choose the "Tools for Team Leaders" installation option, and enter the correct location of the Team Configuration Directory when prompted. See the installation help topic for more information.
    • Simultaneously open your Personal Dashboard and the Team Dashboard on your local computer. (Note: only one team member can have the Team Dashboard open at a time, so some inter-team coordination will be needed.)
    • In the Team Dashboard, use the Script Button menu to open the Team Project Tools and Settings page, then click on the link to join the team project.
    • Walk through the steps in the Team Project Setup Wizard to join the team project.
    • Shut down the Team Dashboard, to give other team members a chance to join.
