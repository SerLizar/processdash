Exporting your data

Presently the dashboard provides two methods of exporting data to other applications.

Creating an ASCII flat file

You can create an ASCII file of your data by choosing Export from the "C" menu. Doing so will bring up the following tool:

NOTE: This tool will be replaced by a much more robust import/export capability in a future version of the dashboard and is to be considered a deprecated feature.

The Export tool presents the project hierarchy in a pane on the left, and options to export either your process data or the hierarchy list.

Exporting data

If you choose to export your data, only the process nodes that have actually measured data will be written to a file in the following format:

/complete/path/to/data/item,<value>
  ...
!Date & Time!,<number of minutes recorded>
  ...

The data elements themselves will not be in any particular order, but the time data will be in chronological order. This format was originally implemented to allow earned value tracking in an external tool and does not really provide a complete dump of the data in the dashboard repository.

Exporting the Hierarchy list

If you choose to export the hierarchy list, the entire hierarchy tree will be written to a file. The file will have the form of one entry per line in the following format:

/complete/path/to/data/item

Exporting data to Excel (PC only)

Many of the reporting forms in the dashboard have links that say "Export to Excel...". When this link is clicked, an Excel sheet will come up that contains the data from that form.

There is a subtle difference between the way that this export is handled in Netscape versus the way it is handled in Internet Explorer.

  • In Netscape, the sheet will have static data elements that contain the values of the process variables at the instant that the export was done.
  • In Internet Explorer, the sheet will be a linked spreadsheet. Excel will connect to the dashboard and retrieve the data items. Excel will typically also display a floating "External data" toolbar, which can be used to refresh the data (as long as the dashboard is still running). By clicking the "Data Range Properties" button on this floating toolbar, Excel can be configured to refresh the data automatically whenever the spreadsheet is opened.

The "Export to Excel..." option will appear on

  • tables that are displayed by clicking on a graphical chart (see Data Charts and Reports for more information)
  • the table that is displayed when you click on the "historical data" link from a PSP1.0 or higher process script
  • the HTML time log
  • the HTML defect log
  • the R3 report
  • the R4 report
