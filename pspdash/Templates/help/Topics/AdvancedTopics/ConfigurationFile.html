Configuration File

The Configuration File is used to customize the behavior of the Process Dashboard.  This file is named "pspdash.ini" on Windows platforms, and ".pspdash" on all other platforms.  This text file can be located in either the user's data directory (default) or in the same directory as the pspdash.jar file (used only if a settings file is not found in the data directory). 

The most common settings and their usage are listed below:

Name Use / Format
stateFile Location of data file used for hierarchy.
Filename format.
Default: the dashboard data directory.
browser.command Command used to bring up HTML browser.
Command-line text.
Default: uses the browser set up during the installation.
templates.directory Directory containing templates.
Path name format.
Default: uses the templates that come with the dashboard and additional templates in the same directory as the pspdash.jar file.
dateFormat Display format for date fields.
Tab separated list, where the first element is used for display (other values can be used for entering dates in other formats). See the Java documentation for the DateFormatter class for date formats.
Default: the Java default for the DateFormatter class.
dateTimeFormat Display format for time and date fields.
See entry for dateFormat.
defectDialog.autostart True to start timer when defect dialog starts.
true or false.
Default: true.
pauseButton.showCurrent Appearance of the pause / play button on the dashboard.
true - shows PLAY symbol while playing (current state)
false - shows PLAY symbol while paused (affect of button press).
Default: false.
probeDialog.onlyCompleted Flag to set implicit filtering of data by the PROBE dialog.
true - PROBE only correlates data from completed projects.
Default: true.
http.allowRemote Allow or deny remote http connections.
false - the dashboard rejects http connections unless they come from "localhost". This prevents other people from viewing your PSP data.
true - accept http connections from other machines. If you want to be able to view your own PSP data in a browser on another machine, set this to "true".
Default: false.
