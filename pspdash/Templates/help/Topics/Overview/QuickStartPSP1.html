Quick start example: a PSP1.1 project

This example will show how to use the Process Dashboard for a PSP1.1 level software project. The example that will be followed is program 5A from the book A Discipline for Software Engineering by Watts Humphrey. This program was chosen because it will show nearly all of the features that the Process Dashboard currently offers without being overly complicated*.

Here are the steps that we will take in this example:

  1. Creating the project
  2. Planning the project
  3. Designing the program
  4. Writing the code
  5. Compiling the program
  6. Testing the program
  7. Postmortem

Next step: Creating the project

*Note: This example demonstrates PSP1.1 only because it shows many features of the Process Dashboard. You should realize, however, that in order to use PSP1.1 (and PROBE in particular) you must have historical data. When you first begin using the dashboard, you won't have any historical data in the dashboard. Therefore:
  • If you have been using PSP for some time, you may be able to use the "PSP Data Import" add-on process set (available for download from http://processdash.sourceforge.net) to enter your historical data into the dashboard.
  • If you have never used PSP before, you should use the PSP0 process for your first project, and the PSP0.1 process for your second and third projects. You should also take a PSP course to learn how to use the PSP processes.
