PROBE Estimating Script

Phase
Number		 Purpose To guide the PROBE estimation process.
Entry Criteria
  • Requirements statement
  • Size Estimating Template and instructions
  • LOC per method data for object types
  • Time Recording Log
1 Conceptual Design
  • Review the requirements and produce a conceptual design to meet them.
2 Identify Objects Using the Size Estimating Template,
  • Identify and name the objects needed to implement this design.
  • Judge the number of methods in each object.
  • Determine the type category for each object (Logic, I/O, Calculation, Text, Data, Set-up).
  • Estimate the relative size of the object's methods (very small, small, medium, large, very large).
  • Find the LOC per method from the object database for that object type.
  • Calculate the estimated object size as the number of methods times the LOC per method.
  • Judge which of these objects will likely be entered in the reuse library and check the box in the LOC(New Reused) column.
3 Other Program LOC
  • Follow the Size Estimating Template instructions to estimate the base, deleted, modified, added, reused, and new reused LOC.
4 Estimating Basis
  • A - If you have no historical data, use your judgment to estimate the new and changed LOC and the required development time per LOC.
  • B - If you have sufficient actual data on development time and program size, estimate the new and changed LOC for the new program and multiply it by your historical new and changed LOC per hour productivity rate.
  • C - If you have sufficient actual development time and actual object LOC data, calculate the B0 and B1 regression factors from these actual object LOC and actual hours data.
  • D - If you have sufficient estimated object LOC data, calculate two sets of B0 and B1 regression parameters. One for the estimated object LOC versus actual new and changed LOC and use these parameters to calcuate the estimated new and changed LOC on the Size Estimating Template. Calculate the second set of B0 and B1 regression parameters for estimated object LOC and actual hours. Use these parameters to estimate total planned development time.
5 Sufficient Data
  • To use the averaging method, it is sufficient to have historical data on actual new and changed LOC and actual development hours for one project.
  • To use the regression method, you must have a minimum of three sets of historical development data.
  • With three or more sets of historical data, the correlation between LOC (estimate or actual) and development time should give an r value such that r2 >= 0.50.
6 Regression Method
  • Use the linear regression procedure described in Appendix A, Table A27, to calculate the B0 and B1 regression parameters.
  • Select the size data as actual object LOC (case C in step 4) or estimated object LOC (case D in step 4).
Note: the regression parameters in the Size Estimating Template relate the base program additions (BA), new object (NO), and base program modifications (M) to total new and changed LOC (N). Your regression dataset should include these same items.
7 Regression Parameter Test The absolute magnitude of B0 should be substantially less than the expected size (for size estimates) or development time (for hour estimates) of the program you are estimating.
  • For size estimates, the value of B1 should be positive and relatively near to 1.0.
  • For development time estimates, the value of 1/B1 should be positive and relatively near to your historical LOC/hour productivity.
If these conditions are not met, use the averaging method.
8 Averaging Method
  • Set the value of B0 = 0.
  • Set B1 = the sum of the development time for each program in the database / the sum of the actual object LOC for each program in the database.
Note: the size data in the size estimating database should include the same items included in the new program estimate.
9 Estimate
  • Estimate the projected LOC in the Size Estimating Template and calculate the development time:
    Hours = B0 + B1 * (projected plus modified LOC).
10 Prediction Interval
  • If you used the regression method, calculate the prediction interval for the estimate as described in Appendix A, section A8.
Note: the use of actual new and changed LOC and actual development time in the regression calculations is equivalent to assuming that the size estimate was exactly correct. This will generally give an unrealistically small prediction interval.

Next: Continue planning.
