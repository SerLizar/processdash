Optional: Create Master Project (Program Leader)

In the Process Dashboard, a Team Project allows data to be rolled up data from several individuals to the team level. For larger projects, it may also be necessary to roll up data from several team projects to the program level. Master Projects are designed to meet this need.

To create a new master project, follow these steps:

  1. Start the Team Dashboard using the appropriately named shortcut on the team leader's computer. The Team Dashboard will start. (If your personal dashboard is also running, you can easily distinguish the Team Dashboard because it displays "Team Dashboard" in the title bar, and displays a different icon for the window.)
  2. On the C menu of the team dashboard, choose the Hierarchy menu item. The Hierarchy Editor window will appear.
  3. In the Hierarchy Editor, highlight a node in the hierarchy such as the top-level Project node. On the Edit menu, choose Add Template → Team Project. A new node called Team Project will be created; give it a short name that describes the master project. Choose File → Save, then File → Close.
  4. Using the dashboard Hierarchy Menus, navigate to the project you just created. Click the dashboard Script Button to display the Team Project Setup Wizard.
  5. The Team Project Setup Wizard will appear. Read each page carefully and supply the information requested. One page of the wizard will ask whether you wish to create a team project or a master project; choose the Master Project option.
  6. At one point, the Team Project Setup Wizard will give you an opportunity to select the directory where you would like to store data for this master project. By default, it will encourage you to store data into the team configuration directory. Most teams are strongly encouraged to keep this default, as it is simpler and less confusing to keep all your data in one place. If you have a compelling need to place master project data in a separate location than configuration data, you can alter this path. However, the directory you choose must meet all of the same accessibility and permissions criteria as the team configuration directory.
  7. When the wizard finishes, you can close the browser window.
