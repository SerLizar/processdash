Configuration File

The Configuration File is used to customize the behavior of the Process Dashboard. This file is named "pspdash.ini" on Windows platforms, and ".pspdash" on all other platforms. This text file can be located in either the user's data directory (default) or in the same directory as the pspdash.jar file (used only if a settings file is not found in the data directory).

All settings in the file are set by lines of the form

<settingname>=<value>

Comments can be placed in the file. Comments are lines that start with the # character.

The settings available in this file and their usage are listed below:

  • autoRealization
    Policy for auto-realization of data. To save time and memory, formulas are not immediately calculated when the dashboard opens. Instead, they are calculated (realized) at some later time. This realization always occurs automatically when their value must be displayed for the first time on a chart, form, or report. In addition, a background process runs when the the dashboard starts, and slowly calculates formulas. If this setting has the value "full", all data will be automatically realized by this background process. If this setting has the value "min", only global (e.g. "To Date") data is auto-realized. If this setting has the value "none", no data will be auto-realized. If you are on a slow computer, and you notice irritating delays the first time you view a chart, form, or report, try using the "full" option. If you experience irritating delays when saving edits to the hierarchy, try using the "min" or "none" options.
    Format:full, min, or none
    Default:min
    Example:autoRealization=full

  • autoUpdate.disabled
    If this property is set to "true", the dashboard will not automatically perform a monthly check for updated versions of the dashboard software or the add-on process sets.
    Format:true or false
    Default:false
    Example:autoUpdate.disabled=true

  • autoUpdate.lastCheckDate
    This is the last date when the dashboard was able to successfully check for updated versions of the dashboard software and add-on process sets. (Measured in milliseconds since Jan 1, 1970.)
    Format:Number
    Default:N/A
    Example:autoUpdate.lastCheckDate=991710921713

  • autoUpdate.proxyPassword
    When running in the context of an auto-redirecting HTTP proxy server, use this password to authenticate.
    Format:Text
    Default:N/A
    Example:autoUpdate.proxyPassword=Cl3verpa55w0rd

  • autoUpdate.proxyUsername
    When running in the context of an auto-redirecting HTTP proxy server, use this username to authenticate.
    Format:Text
    Default:N/A
    Example:autoUpdate.proxyUsername=dashuser

  • autoUpdate.remind
    Controls whether to remind the user about available software updates of the dashboard the next time the dashboard starts, even if a check has been performed recently.
    If true, the dashboard will remind the user.
    If false, the dashboard will not remind the user.
    Format:true or false
    Default:false
    Example:autoUpdate.remind=true

  • browser.command
    Command used to bring up HTML browser.
    Format:a command line
    Default:uses the browser set up during the installation
    Examples:browser.command=/usr/bin/netscape
    browser.command=C:\Program Files\Netscape\Communicator\Program\netscape.exe
    browser.command=C:\Program Files\Internet Explorer\iexplore.exe

  • browser.readonly.color
    In Internet Explorer, read-only data elements on forms will be flagged with this background color.
    Format:text
    Default:grey
    Example:browser.readonly.color=blue

  • dateFormat
    Display format for date fields.
    Format:A list of recognized date formats, separated by the | character. The first format in the list will be used when displaying dates. For valid format syntax, see the documentation for java.text.SimpleDateFormat in the Sun Java Developer Kit.
    Default:The Java default for the DateFormatter class.
    Example:dateFormat=

  • dateTimeFormat
    Display format for date and time fields.
    Format:A list of recognized date formats, separated by the | character. The first format in the list will be used when displaying date-times. For valid format syntax, see the documentation for java.text.SimpleDateFormat in the Sun Java Developer Kit.
    Default:The Java default for the DateFormatter class.
    Example:dateFormat=

  • defectDialog.autostart
    Controls whether to start the defect timer when the defect dialog appears.
    If this property is set to "true", the defect dialog box will come up with the timer already running. Any other value for this property means the user must manually press the "Start Timing" button to begin the timer for fixing the defect.
    Format:true or false.
    Default:true
    Example:defectDialog.autostart=false

  • defectDialog.phaseMap
    Describes common defect injection/removal patterns. Each pair is interpreted to mean "Often defects are injected in phase A, and removed in phase B." When you remove a defect during phase B of a process, it will then use phase A as the default injection phase.
    Format:It should be a list of phase pairs; the pairs should be separated from each other by "|" characters. Each pair should be of the form "phase A=>phase B".
    Default:
    Example:defectDialog.phaseMap=

  • defectDialog.restrictSequence
    If this property is set to true, the defect dialog will check injection and removal phases to ensure that defects are NOT removed before they are injected. (You might want to disable this feature if you make use of iterative processes where the real-life task sequence does not strictly follow the order the of nodes in the hierarchy.)
    Format:true or false
    Default:true
    Example:defectDialog.restrictSequence=false

  • defectTypeStandard
    The name of the default defect type standard.
    Format:Text
    Default:
    Example:defectTypeStandard=

  • excel.exportMethod
    The method to use for exporting data into Excel. If this is set to "mime", tables will be exported to Excel using a mime-type. If this is set to "iqy", tables will be exported using a Microsoft Internet Web Query (.IQY) file. Set this to "none" if you don't have Excel and don't care to see the "Export to Excel" link underneath tables.
    Format:mime, icy, or none
    Default:icy if your browser is Internet Explorer, and mime otherwise.
    Example:excel.exportMethod=none

  • hiddenData
    Data names to be suppressed from user-visible data lists (by default).
    Format:This property should be a series of regular expressions matching data names, separated by the vertical bar character '|'.
    Default:All data is visible
    Example:hiddenData=
  • http.allowRemote
    Controls remote access to the dashboard's built-in webserver.
    If true, the dashboard will allow connections from other machines.
    If false, the dashboard will reject all HTTP connections unless they come from "localhost".
    Format:true or false.
    Default:false
    Example:http.allowRemote=true

  • http.port
    The port number where the dashboard should listen for incoming http connections. (Note: the port one number less than this port - typically 2467 - must be available also, for incoming data connections.)
    Format:Number
    Default:2468
    Example:http.port=3000

  • pauseButton.historySize
    The maximum number of entries which should be retained in the play/pause button's history list.
    Format:Number
    Default:
    Example:pauseButton.historySize=10

  • pauseButton.quiet
    If this property is set to "true", the Play/Pause button will not provide any audible feedback.
    Format:true or false
    Default:false
    Example:pauseButton.quiet=true

  • pauseButton.showCurrent
    Controls the appearance of the play/pause button on the dashboard.
    If true, the button will show the PLAY symbol [] while playing (the button shows its current state).
    If false, the button will show the PLAY symbol while paused (the button shows what it will do if pressed).
    Format:true or false.
    Default:true
    Example:pauseButton.showCurrent=false

  • probeDialog.dataList
    If this property is set to "simple", the PROBE dialog will only display data elements relating to the standard PROBE methods A,B,C, and D. If this setting has any other value, PROBE will display an exhaustive list of data element choices for correlating.
    Format:simple or other text
    Default:simple
    Example:probeDialog.dataList=all

  • probeDialog.onlyCompleted
    Controls implicit filtering of data by the PROBE tool.
    If true, the PROBE tool only correlates data from completed projects.
    If false, the PROBE tool will use all available data.
    Format:true or false.
    Default:true
    Example:probeDialog.onlyCompleted=false

  • stateFile
    Location of the data file that stores the hierarchy.
    Format:complete pathname
    Default:<the dashboard data directory>/state
    Examples:stateFile=/home/username/mypspdata/state
    stateFile=c:\mypspdata\state

  • templates.directory
    Directory that contains the process templates.
    Format:directory path
    Default:uses the templates that come with the dashboard and additional templates in the same directory as the pspdash.jar file.
    Examples:templates.directory=/usr/local/dash/Templates
    templates.directory=C:\Program Files\Process Dashboard\Templates

  • timelog.updateInterval
    The dashboard saves the current time log entry periodically. This setting specifies the number of minutes it should wait between saves.
    Format:Number
    Default:1
    Example:timelog.updateInterval=1
